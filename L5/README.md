Метою даної лабораторної роботи є набуття знань про існуючи

### Завдання 1

Завантажте з бібліотеки MATLAB кілька різних зображень.

У цьому завданні були завантажені тестові зображення з бібліотеки MATLAB
за допомогою функції imread. Для відображення зображень була використана функція imshow. 
Команда figure створює нове вікно для кожного зображення, а функція title додає підпис до нього.

```
I1 = imread('cameraman.tif');
I2 = imread('peppers.png');
I3 = imread('autumn.tif');

imgs = {I1, I2, I3};
n = length(imgs);

figure(1); clf;
for i = 1:n
    subplot(1,n,i);
    imshow(imgs{i});
    title(['Original ', num2str(i)]);
end

```

### Завдання 2

З використанням функції rgb2gray перетворіть кольорові зображення в чорно-білі. Відобразіть результат.

На даному етапі було виконано перетворення кольорових зображень у відтінки сірого за допомогою функції rgb2gray.
Для кожного зображення відображено одночасно оригінал та його чорно-білу версію. Результати згруповані в одному
вікні для зручного порівняння

```
grayImgs = cell(1,n);

figure(2); clf;
for i = 1:n
    
    img = imgs{i};
    
    if size(img,3)==3
        grayImgs{i} = rgb2gray(img);
    else
        grayImgs{i} = img;
    end
    
    subplot(2,n,i);
    imshow(imgs{i});
    title(['Original ', num2str(i)]);
    
    subplot(2,n,i+n);
    imshow(grayImgs{i});
    title(['Gray ', num2str(i)]);
    
end

```

### Завдання 3

З використанням функції dct2 виконайте дискретне косинусне перетворення зображень. 
Відобразіть результат ДКП у вигляді зображення з використанням функції imshow. 
Під час відображенні масиву ДКП-коефіцієнтів використовуйте логарифмічний масштаб.

На даному етапі до кожного зображення було застосовано двовимірне дискретне косинусне перетворення.
Отримані коефіцієнти представлені у логарифмічному масштабі для кращої візуалізації. Це дозволяє
побачити розподіл енергії зображення між низькими та високими частотами. Основна інформація
концентрується у верхньому лівому куті спектра.

```
DCT = cell(1,n);

figure(3); clf;
for i = 1:n
    
    DCT{i} = dct2(grayImgs{i});
    
    subplot(1,n,i);
    imshow(log(abs(DCT{i})+1), []);
    title(['DCT ', num2str(i)]);
    
end
```

7. З використанням функції idct2 відновіть зображення за квантованим
ДКП-спектром для різних значень кроку квантування. Поясніть отримані
результати.

Після квантування спектр ДКП стає більш розрідженим. Більшість високочастотних коефіцієнтів набуває нульових значень. Це свідчить про зменшення кількості інформації та ефективне стиснення.

Після застосування оберненого ДКП було отримано відновлені зображення. При малому значенні квантування (N=10) якість майже не змінюється. При більшому значенні (N=30) зображення стає розмитим, втрачаються дрібні деталі. Це демонструє залежність якості від ступеня квантування.

```
N_values = [10, 30];

for k = 1:length(N_values)
    
    N = N_values(k);
    
    figure(4+k); clf;
    
    for i = 1:n
        
        % квантування
        Jq = N * round(DCT{i} / N);
        
        % спектр
        subplot(2,n,i);
        imshow(log(abs(Jq)+1), []);
        title(['DCT (N=', num2str(N), ')']);
        
        % відновлення
        R = idct2(Jq);
        
        subplot(2,n,i+n);
        imshow(R, []);
        title(['Restored (N=', num2str(N), ')']);
        
    end
end

```

### Завдання 8

Поясніть, яка мета досягається квантуванням коефіцієнтів ДКП.

Квантування потрібне для того, щоб зменшити обсяг даних зображення. Ми просто зменшуємо точність коефіцієнтів ДКП, і маленькі значення часто стають нулями. Через це даних стає менше, і зображення легше зберігати або передавати. Якість трохи погіршується, але зазвичай це не дуже помітно.

### Завдання 9

Чи можливо добитися аналогічної мети й результату, квантуючи
вихідне зображення, а не коефіцієнти його ДКП? Перевірте це на практиці,
квантуючи з використанням процедури I = (round(J/n))*n вихідні
зображення й відобразіть отриманий результат. Поясніть результат.

Я спробувала квантувати не DCT, а саме зображення - просто округлювала значення пікселів. Вийшло, що якість сильно погіршується: видно різкі переходи і картинка виглядає гірше. Це тому, що таке квантування не враховує структуру зображення. Тому квантувати краще саме в області DCT, там втрати менш помітні.

```
n_val = 20;

figure(10); clf;

for i = 1:n
    
    img = grayImgs{i};
    
    Qimg = n_val * round(img / n_val);
    
    subplot(1,n,i);
    imshow(Qimg, []);
    title(['Image quant ', num2str(i)]);
    
end
```

### Завдання 10

Які недоліки ви бачите в стисненні зображень із використанням його ДКП і квантування коефіцієнтів ДКП?

Основним недоліком є втрата якості зображення при сильному квантуванні. 
Можуть з’являтися артефакти, зокрема блокові ефекти та розмиття деталей. 
Метод менш ефективний для зображень з великою кількістю дрібних деталей.

# Висновок

У ході лабораторної роботи було розглянуто основні принципи стиснення зображень та алгоритм JPEG. Особливу увагу приділено дискретному косинусному перетворенню, яке дозволяє перейти від просторового представлення зображення до частотного.

Було встановлено, що після застосування DCT основна частина інформації зосереджується в низькочастотній області, тоді як високочастотні компоненти містять менше значущої інформації. Це дозволяє ефективно виконувати стиснення шляхом їх часткового відкидання.

Також досліджено вплив квантування коефіцієнтів DCT. При збільшенні кроку квантування ступінь стиснення зростає, але якість зображення погіршується через втрату деталей. Додатково перевірено квантування безпосередньо вихідного зображення, яке показало значно гірші результати порівняно з частотною областю.

У результаті підтверджено, що використання DCT та квантування у частотній області є більш ефективним методом стиснення зображень, ніж пряме квантування пікселів.
